# Diagnostics UI

Static HTML viewer for Briscola diagnostics JSON files.

It can display:

- decision logs generated by `scripts/record_diagnostics.py`;
- neural interaction reports generated by `scripts/analyze_neural_interactions.py`.

## Open

From the project root:

```bash
python3.11 -m http.server 8765 --bind 127.0.0.1
```

Then open:

```text
http://127.0.0.1:8765/diagnostics_ui/visual_diagnostics.html
```

Use the file picker to load a diagnostics JSON file.

## Open With A JSON Path

If the JSON file is inside the project folder, it can also be loaded from the
URL:

```text
http://127.0.0.1:8765/diagnostics_ui/visual_diagnostics.html?json=/experiments/results/diagnostics.json
```

## Example JSON Generation

Decision log:

```bash
python3.11 scripts/record_diagnostics.py \
  --checkpoint experiments/results/checkpoint.json \
  --partner perfect \
  --opponents perfect \
  --output experiments/results/diagnostics.json
```

Neural interaction report:

```bash
python3.11 scripts/analyze_neural_interactions.py \
  --checkpoint experiments/results/checkpoint.json \
  --games 100 \
  --output experiments/results/neural_interactions.json
```

## Notes

The UI only visualizes existing JSON files. It does not train, evaluate, or
modify checkpoints.
